# Requirements: Backend Logic Implementation

## Summary

Replace all mock data, placeholder functions, and hardcoded values in the Second Brain cyberpunk UI with fully functional backend logic. The UI is complete with 6 waves of implementation (16 task files). This spec covers: database schema for notes/tags/graph/embeddings, server actions for all CRUD operations, AI-powered semantic search with RAG and citations, Graphify-powered knowledge graph extraction and visualization, persistent user settings, Resend email integration, and comprehensive error handling.

The application is a personal knowledge management system where a single user creates markdown notes, organizes them with tags, visualizes connections through an interactive knowledge graph (auto-generated by Graphify), and queries their knowledge using AI with semantic search over note embeddings.

## Goals

- Replace all mock data arrays with real database queries via Drizzle ORM
- Implement all placeholder functions (console.log, alert, empty onClick) with real server actions
- Build an embedding pipeline that generates vector embeddings for notes and stores them in pgvector
- Implement RAG-based AI query that finds relevant notes via vector similarity and generates answers with citations
- Auto-generate knowledge graph nodes and edges from notes using Graphify
- Persist user settings (theme, system, AI) to the database
- Wire Resend for password reset and email verification emails
- Ensure all navigation links and buttons work correctly
- Add proper error handling, loading states, and optimistic updates

## Non-Goals

- Multi-user support or collaboration features
- Real-time sync or live collaboration
- Mobile app or responsive mobile layout
- OAuth providers (only email/password with Better Auth)
- Payment integration (Polar)
- File/image upload for notes (future scope)
- Full-text search (using vector search instead)

## Acceptance Criteria

- [ ] Creating a note saves it to PostgreSQL and displays it in the notes list
- [ ] Editing a note persists changes and updates the knowledge graph
- [ ] Deleting a note removes it and cleans up related tags/edges/embeddings
- [ ] Tags can be created, edited, deleted with correct usage counts
- [ ] Knowledge graph renders real nodes/edges extracted from notes via Graphify
- [ ] AI query returns answers with real citations linking to actual notes
- [ ] AI chat panel in note editor provides context-aware responses about note content
- [ ] Dashboard shows real stats (note count, node count, connection count) and recent activity
- [ ] Settings changes persist across page reloads
- [ ] Sidebar shows real recent notes and stats
- [ ] Password reset sends actual emails via Resend
- [ ] Email verification sends actual emails via Resend
- [ ] All placeholder navigation links and buttons work
- [ ] Error states display properly with neon-styled error messages
- [ ] Loading states show the LoadingOrb component during async operations

## Assumptions

- PostgreSQL 18 with pgvector extension is running via Docker Compose
- Better Auth is already configured with email/password (existing)
- OpenRouter API key is available for AI chat and embeddings
- Resend API key will be provided for email integration
- The user runs the app locally for personal use
- Graphify CLI is available for knowledge graph extraction
- All UI components exist and compile without errors
- The existing database has 4 Better Auth tables (user, session, account, verification)

## Technical Constraints

- **ORM**: Drizzle ORM with postgres.js driver (existing setup in src/lib/db.ts)
- **Database**: PostgreSQL 18 with pgvector (Docker image pgvector/pgvector:pg18)
- **Server Actions**: Use Next.js Server Actions (no REST API routes for CRUD)
- **AI Provider**: OpenRouter via @openrouter/ai-sdk-provider (existing) for chat; for embeddings use @ai-sdk/openai with OPENAI_API_KEY since OpenRouter may not support embedding endpoints
- **Embedding Model**: text-embedding-3-large (referenced in env.example)
- **Vector Search**: pgvector with cosine similarity for semantic search
- **Auth**: Better Auth already configured — do not modify auth setup, only add email provider
- **Schema**: All new tables use UUID for IDs (not related to Better Auth)
- **Email**: Resend SDK for transactional emails
- **Migration**: Use `drizzle-kit generate` and `drizzle-kit migrate` — NEVER `drizzle-kit push`
- **TypeScript**: Strict mode with noUnusedLocals, noUnusedParameters
- **ESLint**: Follow existing eslint.config.mjs rules including React 19 setState-in-effect
