# Task 01: Database Schema Extension

## Status

complete

## Wave

1

## Description

Extend the existing Drizzle ORM schema (src/lib/schema.ts) with all application tables needed for the Second Brain app. Currently only 4 Better Auth tables exist (user, session, account, verification). This task adds: notes, tags, note_tags (junction), graph_nodes, graph_edges, note_embeddings, ai_queries, and user_settings tables. After modifying the schema, generate and apply the migration.

## Dependencies

**Depends on:** None (Wave 1)
**Blocks:** task-04-notes, task-05-tags, task-06-settings, task-07-embeddings

**Context from dependencies:** The existing schema.ts contains 4 Better Auth tables (user, session, account, verification) with `user.id` as text PK. New tables use UUID for IDs (non-auth tables). The database is PostgreSQL 18 with pgvector extension available in the Docker image (pgvector/pgvector:pg18).

## Files to Create

- `drizzle/` — New migration files generated by drizzle-kit

## Technical Details

### Implementation Steps

1. Open `src/lib/schema.ts` and add the following tables AFTER the existing ones. Keep all existing imports and tables untouched.

2. Add new imports at top of schema.ts:

```typescript
import {
  uuid,
  jsonb,
  integer,
  text,
  timestamp,
  boolean,
  pgTable,
  primaryKey,
  index,
  uniqueIndex,
} from "drizzle-orm/pg-core";
```

Note: Check what's already imported and only add what's missing.

3. Add these tables:

**notes table:**

```typescript
export const notes = pgTable(
  "notes",
  {
    id: uuid("id").defaultRandom().primaryKey(),
    userId: text("user_id")
      .notNull()
      .references(() => user.id, { onDelete: "cascade" }),
    title: text("title").notNull(),
    content: text("content").notNull().default(""),
    excerpt: text("excerpt").notNull().default(""),
    importance: integer("importance").notNull().default(3),
    createdAt: timestamp("created_at", { withTimezone: true }).defaultNow().notNull(),
    updatedAt: timestamp("updated_at", { withTimezone: true })
      .defaultNow()
      .notNull()
      .$onUpdate(() => new Date()),
  },
  (table) => [
    index("notes_user_id_idx").on(table.userId),
    index("notes_updated_at_idx").on(table.updatedAt),
  ]
);
```

4. After modifying schema.ts, generate the migration:

```bash
pnpm run db:generate
```

5. Apply the migration:

```bash
pnpm run db:migrate
```

6. After migration is generated, manually edit the generated SQL file to add the pgvector extension and vector column:

- Add at the top: `CREATE EXTENSION IF NOT EXISTS vector;`
- Add after the note_embeddings table creation: `ALTER TABLE "note_embeddings" ADD COLUMN "embedding_vec" vector(1536);`
- Add an index: `CREATE INDEX "note_embeddings_vec_idx" ON "note_embeddings" USING cosine ("embedding_vec");`

7. Re-run the migration:

```bash
pnpm run db:migrate
```

### Environment Variables

- `POSTGRES_URL` — Must be set to connect to the database

### Important Notes

- Do NOT use `drizzle-kit push` — only `generate` + `migrate`
- The `user` table already exists from Better Auth — reference it with foreign keys, do not recreate it
- All ID columns for new tables use UUID with `defaultRandom()`, NOT text IDs like Better Auth tables
- The `excerpt` field on notes should be auto-populated from the first 200 characters of content (done in server actions, not schema)
- The `embedding` field is stored as text in Drizzle because pgvector types aren't natively supported in the ORM schema definition — the actual vector column is added via raw SQL in the migration
- The `citations` field in ai_queries is JSONB storing an array of `{ noteId, noteTitle, excerpt, relevance }` objects
